# 01_requirements

## Purpose

Holds the approved, written requirements that authorise work in this Mini-AIOS subfolder. A requirement must exist here before any build, query or change is performed elsewhere.

## Accepted Assets

- Approved written requirements with a Requirement ID
- Scope statements (what is in / out of scope)
- Expected evidence definitions for each requirement
- Named reviewer(s) for each requirement
- Measurable PASS / FAIL rules

## Not Allowed

- Unapproved or verbal-only requirements
- Vague goals with no Requirement ID, scope or PASS/FAIL rule
- Business rules, financial logic or PPC logic
- Source code or application configuration

## Source and Evidence Rule

Every important asset stored here must reference its source (where the truth came from) and its evidence (proof the work was done), with the evidence stored in `06_evidence` and linked by path. If an asset cannot point to a source and evidence, it is treated as not done and must not be relied upon.
