# 06_evidence

## Purpose

Holds proof that work was actually performed. Per the operating rule: no evidence = not done. Important assets in other folders must point to evidence stored here.

## Accepted Assets

- Command output and terminal logs
- Git status / diff output (read-only captures)
- Screenshots and exported logs
- Query results captured as evidence files
- Evidence markdown files describing what was proven

## Not Allowed

- Claims of completion with no captured proof
- Edited or fabricated output
- Live secrets or credentials inside evidence files

## Source and Evidence Rule

Every important asset stored here must reference its source (where the truth came from) and its evidence (proof the work was done), with the evidence stored in `06_evidence` and linked by path. If an asset cannot point to a source and evidence, it is treated as not done and must not be relied upon.
