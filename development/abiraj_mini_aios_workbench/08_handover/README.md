# 08_handover

## Purpose

Holds documents that let an unknown developer or another LLM continue this work using only the saved files, with no verbal explanation required.

## Accepted Assets

- Handover notes describing current state and next steps
- Pointers to asset paths, evidence paths and validation paths
- Context another person needs to safely continue

## Not Allowed

- Handovers that depend on undocumented verbal knowledge
- Out-of-date handovers presented as current
- Sensitive credentials or secrets

## Source and Evidence Rule

Every important asset stored here must reference its source (where the truth came from) and its evidence (proof the work was done), with the evidence stored in `06_evidence` and linked by path. If an asset cannot point to a source and evidence, it is treated as not done and must not be relied upon.
