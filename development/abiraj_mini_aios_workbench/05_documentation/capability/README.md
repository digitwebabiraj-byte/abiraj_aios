## Purpose

Holds reusable extracted methods and capabilities. Maps the Technical Setup Guide's 'Reusable extracted methods and capabilities' output type into the numbered architecture.

## Accepted Assets

- Reusable methods extracted from completed work
- Capability write-ups describing how to reuse an approach
- Generalised techniques pending wider-reuse review

## Not Allowed

- Parent-AIOS promotion (candidates go to 11_parent_aios_candidates)
- Task-specific evidence (belongs in 06_evidence)
- Unsourced claims with no link to the work they came from

## Source and Evidence Rule

Every important asset stored here must reference its source (where the truth came from) and its evidence (proof the work was done), with the evidence stored in `06_evidence` and linked by path. If an asset cannot point to a source and evidence, it is treated as not done and must not be relied upon.
