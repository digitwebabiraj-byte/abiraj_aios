# 10_daily_work

## Purpose

Holds the daily operating records: the day's requirements, work logs, priorities, reusable-asset scans and end-of-day closure references.

## Accepted Assets

- Daily requirements and priorities
- Daily work logs
- Reusable-asset scan notes for the day
- End-of-day closure references

## Not Allowed

- Permanent reusable documentation (belongs in 05_documentation)
- Final validated assets (belong in their target folder)
- Vague notes with no date or task reference

## Source and Evidence Rule

Every important asset stored here must reference its source (where the truth came from) and its evidence (proof the work was done), with the evidence stored in `06_evidence` and linked by path. If an asset cannot point to a source and evidence, it is treated as not done and must not be relied upon.
