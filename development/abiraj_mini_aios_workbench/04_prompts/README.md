# 04_prompts

## Purpose

Holds the approved GPT-generated Claude Code prompts that were actually executed, so every action taken can be traced to the instruction that authorised it.

## Accepted Assets

- Approved GPT discovery prompts
- Approved GPT implementation prompts
- Approved GPT validation, documentation and closure prompts

## Not Allowed

- Unapproved, draft or experimental prompts presented as approved
- Prompts that request work outside the assigned subfolder
- Prompts that invent missing requirements

## Source and Evidence Rule

Every important asset stored here must reference its source (where the truth came from) and its evidence (proof the work was done), with the evidence stored in `06_evidence` and linked by path. If an asset cannot point to a source and evidence, it is treated as not done and must not be relied upon.
